# Chatter — private 1-to-1 messaging (PHP + SQLite3)

A small private messaging app, styled like LINE/SMS: an inbox of your
conversations, and a one-to-one chat thread that only the two
participants can see. Built with **SQLite3** instead of MySQL.

## Features / where to find them

| Requirement                     | File(s)                                   |
|-----------------------------------|---------------------------------------------|
| Database connection               | `config.php`                                |
| Create (Insert)                    | `create.php` (send-message handler)         |
| Read (Display)                     | `index.php` (inbox), `chat.php` (thread)    |
| Update (Edit)                      | `edit.php`                                  |
| Delete                             | `delete.php`                                |
| Basic form validation              | `create.php`, `edit.php`, `register.php`, `login.php` |
| Upload/display images (optional)   | `create.php`, `edit.php`, `chat.php`        |
| Navigation menu                    | `includes/header.php`                       |
| User login/logout (optional)       | `login.php`, `logout.php`, `register.php`, `auth.php` |

## How the private messaging works

- Every message row in the database has a **sender_id** and a
  **recipient_id** (see `messages` table in `config.php`).
- `chat.php` only ever queries messages where the logged-in user is
  *either* the sender *or* the recipient of that specific
  conversation (`chat.php?with=<user id>`) — nobody else's messages
  can appear on that page, even if they guess the URL, because the
  `WHERE` clause is scoped to exactly those two user IDs.
- `index.php` (the inbox) lists only the conversations the logged-in
  user is actually part of, with the latest message as a preview.
- `new_chat.php` lets you pick any other registered user to start a
  new private conversation with.
- `edit.php` / `delete.php` also check that the logged-in user is the
  **sender** of a message before allowing changes — so you can only
  edit or delete your own messages, never someone else's.

## ⚠️ If you already ran the app before this update

The database schema changed (the old public `chatters` table became a
private `messages` table with sender/recipient). If you have an
existing `database/chatter.db` file from testing the earlier public
version, delete it before running the app again:

```bash
rm database/chatter.db
```

A fresh one will be created automatically on the next page load.

## Requirements

- PHP 7.4+ with the **sqlite3** extension enabled.
- No MySQL server needed — SQLite3 stores everything in a single file
  at `database/chatter.db`, created automatically the first time the
  app runs.

## Running it locally

```bash
cd chatter-app
php -S localhost:8000
```

Then open http://localhost:8000. Register at least two accounts (in
two different browsers, or one normal + one incognito window) to try
out a real private conversation between them.

## How the pieces fit together (for your presentation)

- **PHP** is the backend: `config.php` opens the SQLite connection and
  creates tables, the CRUD logic lives in `create.php` / `chat.php` /
  `edit.php` / `delete.php`, and `auth.php` handles sessions for
  login/logout and who's allowed to see what.
- **HTML** is generated by PHP using `<?= ?>` / `<?php ?>` tags mixed
  into the page files — e.g. `chat.php` loops over the two users'
  shared messages and prints one bubble per message.
- **CSS** (`assets/style.css`) is linked once in `includes/header.php`
  and gives the app its LINE/SMS-style bubble look — sent messages in
  green on the right, received messages in gray on the left.
- **JavaScript** (`assets/script.js`) runs in the browser after the
  page loads, and auto-scrolls the chat thread to the newest message,
  plus a live character counter on the edit form.

### The CRUD flow, in one sentence each

- **Create**: the composer on `chat.php` submits to `create.php`,
  which validates the message/recipient, optionally saves an uploaded
  image, then `INSERT`s the row with `sender_id` = you and
  `recipient_id` = the person you're chatting with.
- **Read**: `index.php` lists your conversations; `chat.php` runs a
  `SELECT` scoped to exactly the two participants, ordered oldest to
  newest.
- **Update**: `edit.php` loads the message, checks you're the sender,
  then runs an `UPDATE` after validation.
- **Delete**: `delete.php` checks you're the sender, then runs a
  `DELETE` and removes the uploaded image file if there was one.

All database access uses **prepared statements** to avoid SQL
injection, and all output goes through the `h()` helper
(`htmlspecialchars()`) to avoid XSS.

## Notes

- Passwords are hashed with `password_hash()` / verified with
  `password_verify()` — never stored in plain text.
- Only the sender of a message can edit or delete it.
- A conversation is private to its two participants — no admin or
  "public feed" view exists in this app by design.
